# Plan Before Building

A few minutes of planning prevents wandering, rework, and half-finished changes.

## Steps

1. Restate the goal in one sentence — what does "done" look like?
2. List the files you will create or change, and the steps in order.
3. Note unknowns and assumptions — anything you would want to confirm or might be wrong about.
4. Pick the smallest first slice that delivers something testable.
5. Proceed, adjusting the plan as you learn.

## Rules

- For any task with more than ~3 steps, plan briefly before writing code.
- Keep the plan short — a handful of lines, not a document.
- Prefer the smallest working slice over a big-bang change.
- If the goal is ambiguous, resolve it before building, not after.
