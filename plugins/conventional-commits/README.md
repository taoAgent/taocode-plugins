# Conventional Commits

A skill that shapes git commit messages to the [Conventional Commits](https://www.conventionalcommits.org) spec.

## What's inside

- **`conventional-commits`** (model-invoked) — activates when you're writing a commit message and
  formats it as `type(scope): summary` with an optional body and footer (`feat`, `fix`, `docs`,
  `refactor`, …), including `BREAKING CHANGE:` footers.

## Why

Consistent, parseable history — readable logs, and changelog / semver tooling that can derive
releases straight from commit types.
