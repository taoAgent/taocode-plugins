# Frontend Design

Make the interface look intentional, not like an untouched framework default.

## Steps

1. Establish a clear visual hierarchy — decide what the eye should hit first, second, third, and use size, weight, and placement to enforce it.
2. Pick a deliberate type scale and a small set of weights; apply them consistently instead of ad-hoc font sizes.
3. Use one consistent spacing scale (e.g. 4/8px steps) for margins, padding, and gaps.
4. Choose a restrained, purposeful color palette — a few intentional colors, not the framework's default blue on white.
5. Design every state, not just the happy one: hover, focus, active, disabled, empty, loading, and error.
6. Avoid templated defaults — the centered card on a gradient, unstyled inputs, generic shadows — unless they genuinely fit.

## Rules

- Every visual choice should be intentional and serve a real voice or brand — not a leftover default.
- Consistency beats novelty: reuse the same scales and tokens everywhere.
- Check color contrast and visible focus states for accessibility.
